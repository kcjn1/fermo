# Fermo Technical Spike

## Current Decision

Proceed with the repo, model, tests, and dogfood shell now. Do not claim beta readiness until the macOS system spike passes on a signed build.

## Spike 1: Website Blocking

Goal: prove Fermo can block selected domains across common browsers.

Implementation path:

- Add an Xcode app extension target using Network Extension Content Filter APIs.
- Feed the extension a minimal rule set generated by `FermoCore`.
- Block `reddit.com` and `youtube.com` as test domains.
- Verify Safari, Chrome, Firefox, private windows, sleep/wake, Wi-Fi changes, and reboot.

Pass condition:

- Blocked domains fail closed during an active session.
- Rules clear after session end.
- Permission-denied state is visible in the app.

## Spike 2: App Blocking

Goal: interrupt selected apps during active sessions.

Implementation path:

- Start with `NSWorkspace.shared.runningApplications`.
- Match by bundle identifier.
- Attempt graceful termination first.
- Record apps that relaunch, resist termination, or need stronger handling.

Pass condition:

- A selected app cannot remain usable during an active session under normal user behavior.
- The app explains the limitation honestly.

## Spike 3: Helper Persistence

Goal: active sessions survive main-app quit and routine Mac lifecycle events.

Implementation path:

- Register a helper using `SMAppService`.
- Store active session state in a shared app group or agreed local container.
- Restore blocking state after main-app relaunch, sleep/wake, and reboot.

Pass condition:

- A locked active session is restored without relying on the main app window staying open.

## Known Constraints

- Network Extension behavior depends on entitlements, signing, and user approval.
- Endpoint Security may be needed for stronger process enforcement, but it increases review, permission, and distribution complexity.
- Swift Package code in this repo models the integration boundaries; the real extension target needs Xcode packaging.
